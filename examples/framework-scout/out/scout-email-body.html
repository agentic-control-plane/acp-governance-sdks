Framework Scout roundup for 2026-04-23. Live web search was unavailable for this run, so this batch is compiled from the known trajectory of frameworks that have been shipping heavily and are most likely to have had material updates in the last ~14 days. Each entry is framed against current ACP adapter coverage (TS: governance core + Anthropic; Python: governance core + CrewAI + LangChain).

Top candidates to evaluate for an ACP adapter:

Framework Lang Why it matters to ACP
LangGraph Py + TS Graph-based orchestration is now the default path for LangChain users running production agents. An acp-langgraph hook that attaches to node transitions would extend our existing acp-langchain package to the serious multi-step workloads.
OpenAI Agents SDK Py + TS Official OpenAI agent SDK with handoffs, guardrails, tracing. High enterprise adoption and already has a guardrail extension point that maps cleanly onto ACP's policy evaluator — probably the single highest-leverage adapter we don't yet have.
Pydantic AI Python Typed agent framework from the Pydantic team; fastest-growing "serious Python" agent stack. Its typed tool/output model fits neatly with ACP’s schema-level policy checks.
Mastra TypeScript TS-first agent framework (workflows, RAG, evals) with a visible YC-era momentum curve. Closes an obvious gap in our TS coverage beyond the Anthropic SDK primitive.
Vercel AI SDK TypeScript Massive TS developer reach. A thin wrapper around generateText/streamText tool-calling loops would put ACP in front of a huge fraction of non-framework TS agents — arguably the widest blast radius of anything on this list.
Google ADK Python Google’s open-source Agent Development Kit — enterprise-relevant for Vertex / Gemini shops where governance asks tend to be explicit. Worth a reconnaissance adapter even if adoption outside GCP is still nascent.
Inngest AgentKit TypeScript Durable-execution agent runtime. Interesting not for volume but for what it governs: long-running, resumable agents are exactly where audit trails + policy enforcement are most valuable.

Suggested next step: pick 1–2 of the top three (LangGraph, OpenAI Agents SDK, Pydantic AI) to prototype adapters against. The OpenAI Agents SDK guardrail hook in particular looks like a 1-day integration for the governed-hook pattern we already use.

Caveat: this run did not hit the live web, so treat it as a prioritization list rather than a "shipped in the last 14 days" ledger. Re-running Scout with WebSearch enabled will let it pin each entry to a concrete release date.
